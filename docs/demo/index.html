单文件 vs 散落文件

Protobuf — 多文件散落

user.proto
types.proto
common.proto
service.proto
agent 需要解析 import 链...

ByteMsg — 单文件 .bmsg

user.bmsg
一切在此，无需 import
// user.proto
import "types.proto";
import "common.proto";

message User {
  uint32 id = 1;
  string name = 2;
}
// user.bmsg — 一切在此
schema: bymsg/v1
package: com.example

enum UserType {
  ADMIN = 0
  USER = 1
}

message User {
  uint32 id = 1 // "ID" | "ID"
  string name = 2 // "名称" | "Name"
}

二进制编码原理

Varint 编码：小数字用更少字节

值 = 1:
1 字节
值 = 300:
2 字节
值 = 16384:
3 字节

核心特性

📄

单文件 .bmsg

一个文件包含所有定义，无需 import，Agent 友好

🗜️

紧凑二进制

Varint + Zigzag 编码，比 JSON 小 50%+

🌐

多语言生成

Go / C# / Java / TypeScript / Python 一键生成

🗺️

原生 Map/List

深度嵌套的 map<K,V> 和 list<T> 一等公民支持

🌍

i18n 内置

代码注释和 CLI 输出自动中英文切换

🔌

插件化
