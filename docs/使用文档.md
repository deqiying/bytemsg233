# bytemsg233 使用文档

## 核心设计哲学

**扁平化 · 无需前向定义 · 写了即用 · 禁止嵌套**

bytemsg233 的 message 是扁平的"平铺类"。不支持嵌套类型定义，所有 message 互相独立。引用其他 message 只需写 id 字段，无需声明类型。

```
# ✅ 正确
message Hero {
    uint32 id = 1
    uint32 weapon_id = 3       // 通过 id 引用
    list<uint32> skill_ids = 4 // id 列表引用
}

# ❌ 错误 — 不允许嵌套
message Hero {
    Weapon weapon = 1           // 不允许
    list<Skill> skills = 2      // 不允许
}
```

## 快速开始

```bash
# 创建 schema
bytemsg233 init myapp

# 编译为多语言
bytemsg233 compile myapp.bmsg --lang go --lang csharp --lang typescript --lang java --lang python

# 中文注释
bytemsg233 compile myapp.bmsg --lang go --locale zh -o ./src
```

## 文档边界规范

- `README.md` 和 `docs/*.md` 面向开发者、维护者、协作者。

## 生成代码约定

### 注释输出

- message 描述会生成类 / 结构体注释。
- field 描述会生成字段注释。
- `--locale zh` 输出中文注释，默认输出英文注释。

### 对象池输出

所有目标语言现在都原生生成对象池辅助接口：

- Go：`Acquire<Type>()` / `Release<Type>()` / `(*Type).Reset()`
- C#：`Type.Rent()` / `Type.Return()` / `instance.Release()` / `instance.Reset()`
- Java：`Type.acquire()` / `Type.release()` / `instance.reset()`
- TypeScript：`Type.acquire()` / `instance.release()` / `instance.reset()`
- Python：`Type.acquire()` / `instance.release()` / `instance.reset()`

默认 reset 会清空旧字段值，避免复用对象时残留脏数据。

### Enum 原生适配

生成器不会把 enum 当“只是一个数字别名”来处理，而是会尽量贴近目标语言习惯：

- Go：生成 `String()`、`Parse<Type>()`、`IsValid()`
- C#：生成 `FromValue()`、`IsDefinedValue()`
- Java：生成 `fromValue()`、`isDefined()`
- TypeScript：生成 `enum` 自身以及 `namespace fromValue()`
- Python：生成 `IntEnum` 和 `from_value()`

目标是让业务层始终用 enum 本身读写字段，而不是回退到裸整数。

## Schema 语法 (.bmsg)

### 基本结构

```
schema: bymsg/v1
package: com.example.game

enum HeroState {
    IDLE = 0
    MOVING = 1
    ATTACKING = 2
    DEAD = 3
}

message Hero {
    uint32 id = 1
    string name = 2
    uint32 level = 3
    uint32 weapon_id = 4
    list<uint32> skill_ids = 5
    map<string, string> metadata = 6
    HeroState state = 7
}
```

### 支持的类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `bool` | 布尔 | `bool active = 1` |
| `int32` / `int64` | 有符号整数 | `int32 score = 1` |
| `uint32` / `uint64` | 无符号整数 | `uint32 id = 1` |
| `float32` / `float64` | 浮点数 | `float32 x = 1` |
| `string` | 字符串 | `string name = 1` |
| `bytes` | 二进制 | `bytes data = 1` |
| `list<T>` | 列表 | `list<string> tags = 1` |
| `map<K,V>` | 字典 | `map<string, uint32> counts = 1` |
| `Enum` | 枚举引用 | `HeroState state = 1` |

### i18n 注释

```
message Hero {
    uint32 id = 1 // "英雄ID" | "Hero ID"
    string name = 2 // "英雄名" | "Hero Name"
    uint32 level = 3 // "等级" | "Level"
}
```

### 支持的枚举值

```
enum Quality {
    WHITE = 0
    GREEN = 1
    BLUE = 2
    PURPLE = 3
    ORANGE = 4
    RED = 5
}
```

## 自定义类型扩展

bytemsg233 支持自定义类型映射，覆盖默认的类型转换。在 `.bmsg` 文件中使用 `@type` 注解：

```
schema: bymsg/v1
package: com.example.game

@type uint64 = long        # Go: int64, Java: long
@type float32 = float      # 保留默认
@type string = text        # 自定义别名

message Hero {
    uint64 id = 1           # 使用自定义映射
    text name = 2
}
```

### 语言特定覆盖

```
@type uint32[go] = uint     # 仅 Go 使用 uint
@type uint32[csharp] = uint # 仅 C# 使用 uint
@type string[ts] = string   # TypeScript 默认
@type string[python] = str  # Python 默认
```

### 别名定义

```
@alias Timestamp = uint64
@alias JsonStr = string
@alias PlayerId = uint32

message Event {
    Timestamp time = 1
    PlayerId player_id = 2
    JsonStr payload = 3
}
```

## YAML 格式 (.bmsg.yaml)

也支持 YAML 格式，适合需要更复杂配置的场景：

```yaml

messages:
  Hero:
    fields:
      id:
        type: uint32
        tag: 1
        description:
          zh: "英雄ID"
          en: "Hero ID"
      name:
        type: string
        tag: 2
      level:
        type: uint32
        tag: 3
      weapon_id:
        type: uint32
        tag: 4
      skill_ids:
        type: "list<uint32>"
        tag: 5
      attrs:
        type: "map<string, string>"
        tag: 6

enums:

      MOVING: 1
      ATTACKING: 2
      DEAD: 3
```

## 为什么不用 Protobuf？

| 特性 | Protobuf | bytemsg233 |
|------|----------|------------|
| 嵌套定义 | 支持（需前向声明） | 禁止（扁平化） |
| Schema 文件 | 多个 `.proto` + import | 单个 `.bmsg` |
| Agent 友好 | 需要解析 import 链 | 一个文件搞定 |
| map 支持 | v3 支持但有限制 | 原生一等公民 |
| i18n 注释 | 不支持 | 内置中英文 |
| 代码生成 | protoc + 插件 | 单二进制 CLI |
| 安装 | 需要 protoc | 一行 curl |
