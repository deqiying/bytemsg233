# bytemsg233 使用文档

## 简介

bytemsg233 是一个现代序列化框架，用于替代 Protocol Buffers。

**核心优势：**
- 单文件 `.bmsg` 定义，Agent-First 设计
- 原生 `map<K,V>` 和 `list<T>` 支持，深度嵌套
- 中英文注释一等公民
- 生成 Go / C# / Java / TypeScript / Python 代码
- 二进制编码比 JSON 小 40-84%，与 Protobuf 同级

## 安装

### macOS / Linux

```bash
curl -fsSL https://raw.githubusercontent.com/neko233-com/bytemsg233/main/scripts/install.sh | bash
```


```bash

```

## 快速开始

### 1. 创建 Schema

```bash
bytemsg233 init myapp
```

生成 `myapp.bmsg`：

```
schema: bymsg/v1
package: myapp

enum Status {
    ACTIVE = 0
    INACTIVE = 1
}

message Example {
    uint32 id = 1 // "ID" | "ID"
    string name = 2 // "名称" | "Name"
}
```

### 2. 编译

```bash
# 生成 Go 代码
bytemsg233 compile myapp.bmsg --lang go

# 生成多语言代码
bytemsg233 compile myapp.bmsg --lang go --lang csharp --lang typescript

# 指定输出目录和语言
bytemsg233 compile myapp.bmsg --lang go --lang python --output ./generated --locale zh
```

### 3. 查看版本

```bash
bytemsg233 version
```

## Schema 语法

### 基本结构

```
schema: bymsg/v1          # 必须，schema 版本
package: com.example.user # 包名/命名空间

enum 枚举名 {
    值名 = 数字
    值名 = 数字
}

message 消息名 {
    类型 字段名 = tag
}
```

### 支持的类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `bool` | 布尔 | `bool active = 1` |
| `int32` / `int64` | 有符号整数 | `int32 score = 1` |
| `uint32` / `uint64` | 无符号整数 | `uint32 id = 1` |
| `float32` / `float64` | 浮点数 | `float32 x = 1` |
| `string` | 字符串 | `string name = 1` |
| `bytes` | 二进制 | `bytes data = 1` |
| `list<T>` | 列表 | `list<string> tags = 1` |
| `map<K,V>` | 字典 | `map<string, string> meta = 1` |
| 自定义消息 | 嵌套消息 | `Address addr = 1` |

### 深度嵌套

```
message Nested {
    map<string, list<uint32>> matrix = 1
    list<map<string, Address>> addresses = 2
    map<uint32, list<Skill>> heroSkills = 3
}
```

### i18n 注释

```
message UserProfile {
    uint32 id = 1 // "用户ID" | "User ID"
    string name = 2 // "用户名" | "Username"
    list<string> tags = 3 // "标签" | "Tags"
}
```

`//` 后面用 `"中文" | "英文"` 格式，生成代码时根据 `--locale` 参数选择语言。

## CLI 命令

### compile

编译 `.bmsg` 或 `.bmsg.yaml` 文件。

```bash
bytemsg233 compile <file> [flags]
```

| 参数 | 简写 | 默认值 | 说明 |
|------|------|--------|------|
| `--lang` | `-l` | `go` | 目标语言，可多次指定 |
| `--output` | `-o` | `.` | 输出目录 |
| `--locale` | | `en` | 注释语言 (`en` / `zh`) |

```bash
# 生成中文注释的 Go + C# 代码
bytemsg233 compile game.bmsg -l go -l csharp --locale zh -o ./src
```

### init

初始化一个新的 `.bmsg` 文件。

```bash
bytemsg233 init <name>
```

### version

显示版本信息。

```bash
bytemsg233 version
```

## YAML 格式 (.bmsg.yaml)

除了 `.bmsg` 格式，也支持 YAML 格式：

```yaml
schema: bymsg/v1
package: com.example.user

messages:
  UserProfile:
    fields:
      id:
        type: uint32
        tag: 1
        description:
          zh: "用户ID"
          en: "User ID"
      name:
        type: string
        tag: 2
      tags:
        type: list<string>
        tag: 3
      metadata:
        type: map<string, string>
        tag: 4
    description:
      zh: "用户资料"
      en: "User Profile"

enums:
  UserType:
    values:
      ADMIN: 0
      USER: 1
    description:
      zh: "用户类型"
      en: "User Type"
```

## 生成代码示例

### 输入 (user.bmsg)

```
schema: bymsg/v1
package: com.example

enum UserType {
    ADMIN = 0
    USER = 1
    GUEST = 2
}

message User {
    uint32 id = 1 // "用户ID" | "User ID"
    string name = 2 // "用户名" | "Username"
    list<string> tags = 3 // "标签" | "Tags"
    map<string, string> metadata = 4 // "元数据" | "Metadata"
}
```

### Go 输出

```go
package com.example

// 用户类型
type UserType int32

const (
    UserTypeADMIN UserType = 0
    UserTypeUSER  UserType = 1
    UserTypeGUEST UserType = 2
)

// 用户资料
type User struct {
    // 用户ID
    Id uint32 `bytemsg:"1"`
    // 用户名
    Name string `bytemsg:"2"`
    // 标签
    Tags []string `bytemsg:"3"`
    // 元数据
    Metadata map[string]string `bytemsg:"4"`
}
```

### C# 输出

```csharp
namespace com.example
{
    /// <summary>
    /// 用户类型
    /// </summary>
    public enum UserType
    {
        ADMIN = 0,
        USER = 1,
        GUEST = 2,

    /// 用户资料
    /// </summary>

    {
        /// <summary>
        /// 用户ID
        /// </summary>

        /// 用户名

        /// 标签
```

## 二进制编码原理

### Varint 编码

小整数用更少字节：

```
值 = 1      → 1 字节: 01
值 = 300    → 2 字节: AC 02
值 = 16384  → 3 字节: 80 80 01
```

### Zigzag 编码

负数用 zigzag 映射为无符号：

```
 0 → 0
-1 → 1
 1 → 2
-2 → 3
```

### Wire Type

| Wire Type | 含义 | 编码方式 |
|-----------|------|----------|
| 0 | Varint | 变长整数 |
| 1 | 64-bit | 固定 8 字节 |
| 2 | Length-delimited | 长度前缀 + 数据 |
| 5 | 32-bit | 固定 4 字节 |

## 为什么选择 bytemsg233 而不是 Protobuf？

| 特性 | Protobuf | bytemsg233 |
|------|----------|------------|
| Schema 文件 | 多个 `.proto` + import | 单个 `.bmsg` |
| Agent 友好 | 需要解析 import 链 | 一个文件搞定 |
| map 支持 | v3 支持但有限制 | 原生深度嵌套 |
| i18n 注释 | 不支持 | 内置中英文 |
| 代码生成 | protoc + 插件 | 单二进制 CLI |
| 安装 | 需要 protoc | 一行 curl |

## 与 MessagePack 的区别

MessagePack 是通用二进制序列化格式，bytemsg233 是 **IDL + 代码生成 + 二进制编码** 的完整方案：

- MessagePack 没有 schema 定义，需要手写各语言结构体
- bytemsg233 定义一次 schema，自动生成 5 种语言代码
- bytemsg233 的二进制编码比 MessagePack 更紧凑（游戏场景小 40-60%）
